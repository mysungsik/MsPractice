============================================================================================================= 1) git push [리모트저장소이름] [브랜치이름] ex) git push origin master *리모트 저장소 이름은 역시나 대부분 [origin]이며 얼마든지 바꿔도 된다. *Branch 이름은 초기는 [master] * git branch -m [브랜치이름] 으로 변경 가능하다 * 변경시에는 [반드시 브랜치 안에 있어야한다] * 브랜치의 이동은 * git checkout [브랜치이름] * 브랜치의 복사 * git checkout -b [브랜치이름] * 브랜치의 병합 * git merge [브랜치이름] * 병합은 [현 위치의 브랜치] 에 [브랜치이름] 을 가진 브랜치를 집어넣는 행위이다. * 현 위치의 브랜치가 어디인지 반드시 확인할 것 [처음 푸쉬를 했을 경우] 2) 푸쉬를 했으면 자동접속 무언가가 뜰 텐데 누르면 확장으로 연결되니 일단 취소를 누른다. 3) 취소를 했으면 상단에 유저네임 치는 것이 나올텐데 내 깃허브 유저네임을 쳐준다(나의 경우는 mysungsik) 4) 비밀번호를 치라고 나오면 [ 토큰 주소를 복사해서 붙여넣는다. ] 5) 완료 [3. 저장소에서 가져오기] ============================================================================================================= 1) git pull ============================================================================================================= 1) 커밋의 삭제 1. [git reset] ==>> 최근 커밋(HEAD 커밋)을 되돌림(삭제) 2. [git reset --hard HEAD~1] ==>> 최근부터(HEAD) 만들어진 [한개의] 커밋을 완전히 삭제 3.
